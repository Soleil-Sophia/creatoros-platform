Route:
Create a new internal route:

/app/labs/authority-engine

Purpose:
The screen should let a user turn technical documentation into structured authority content assets.

Core flow:
1. Load example source or paste a technical source.
2. Define audience, positioning, target platforms, content goals, offer, CTA goal, calendar week, and series name.
3. Analyze source.
4. Show structured analysis:
   - Key Points
   - Excluded Noise
   - System Insight
   - Detected Friction
   - Anti-Pattern
   - Metric Lens
   - Content Angles
5. Generate content assets:
   - Awareness post
   - Authority thread
   - Trust post
   - Education short-form script
   - Conversion CTA
6. Show Quality Gate checklist.
7. Show Content Calendar entries.
8. Allow copy/export of the full run as Markdown.
9. Save, reopen, and delete runs using localStorage.
