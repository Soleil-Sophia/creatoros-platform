#### CML-001 — `src/core/` Subdomain Parity

| Field | Value |
|---|---|
| **Source** | `insight` (reference) + `platform` (canonical) |
| **Target Module** | Platform shared kernel |
| **Action** | VERIFY |
| **Source Path** | `creatoros-insight/src/core/` |
| **Platform Path** | `creatoros-platform/src/core/` |

**Description:** The `src/core/` directory contains 13+ subdomains: `ai`, `assets`, `benchmark`, `blueprint`, `brand`, `content`, `dev`, `hashing`, `instagram`, `intelligence`, `library`, `lineage`, `persistence`, `quality`, `registry`, `validation`. The platform version is canonical per ADR-002 §8. This item verifies that no capability present in insight's core is missing from platform's core.

**Phase A Result:** PASS. All 17 entries present on both sides with matching file counts. Recursive diff found exactly two divergent files, both cases where platform is intentionally ahead of insight, not behind:
- `core/hashing/hashing.ts` — insight's `HashAlgorithm` type is `'sha256' | 'xxhash64'`, but the implementation only ever computes djb2. Platform corrected the type to `'djb2'` to match actual behavior.
- `core/validation/validation.ts` — platform wraps `rule.validate(value)` in try/catch, converting a thrown rule exception into a structured `ValidationViolation` instead of crashing. Insight lacks this.

Do not backport insight's versions of these two files — platform's are the corrected/hardened versions.

**Dependencies:** None. This is a prerequisite for all other items.

**Validation Gates:**
- [x] File-level diff of `insight/src/core/` vs `platform/src/core/` produces no unexplained gaps — PASS, only the two intentional improvements above
- [x] All exported types and functions in insight's core are present and type-compatible in platform's core — PASS
- [ ] TypeScript compilation of `creatoros-platform` passes with zero errors after verification — **PARTIAL**. `vite build` completes with zero errors (1696 modules transformed). However, no `tsconfig.json`, local `tsc` binary, or dedicated typecheck script exists in this repository, so a full standalone TypeScript program-wide typecheck is **NOT EXECUTABLE** with current repository tooling. `vite build`'s transpilation success is not equivalent to a full `tsc --noEmit` pass. Gate status: **PARTIAL** — do not represent this as a full type-compilation pass until a typecheck script is added.

#### CML-004 — BrandProfileExtended Storage

| Field | Value |
|---|---|

| **Action** | MIGRATE |

| **Platform Path** | Does not exist |

**Description:** Extended BrandProfile persistence layer. Key functions:

- `writeBrandProfileExtended(p): BrandProfileExtended` — writes to extended key AND mirrors v1 fields to `creatoros-brand-profile-v1`

**Dependencies:** CML-003 (BrandProfileExtended type must exist first).

**Validation Gates:**

- [ ] `completionPercent` returns 0–100 range correctly
