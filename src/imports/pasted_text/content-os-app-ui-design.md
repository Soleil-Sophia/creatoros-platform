It helps creators, coaches and small teams turn ideas, offers and expertise into structured hooks, scripts, captions, content plans and reusable brand-aligned outputs.

SURFACE / DEPTH RULES

Build the app in 3 visual layers:

1. Background
- deep dark OS backdrop
- subtle tonal gradient
- very controlled atmosphere
- almost no decorative glow
- stable and quiet

2. Midground
- large layout planes
- navigation shell
- content zones
- dashboard panel groups
- tonal separation instead of obvious borders

3. Foreground
- active panels
- result blocks
- primary buttons
- inputs
- key controls
- featured content blocks

SEPARATION RULES
Avoid standard visible borders everywhere.

Use:
- tonal contrast
- top-edge light
- subtle inset / directional shadow
- hard panel splits
- masked segmentation
- spacing and depth
- panel overlap where useful

SHADOW / LIGHT RULES
- directional shadow only
- light edges on important surfaces
- almost no ambient glow except small key focus moments
- no fuzzy haze
- app should feel precise, not dreamy

PANEL / CONTAINER STYLE
- framed but almost borderless
- modular
- architectural
- compact and efficient
- more square than soft SaaS
- panel-within-panel logic
- OS-like surface hierarchy

OUTPUT RULE
